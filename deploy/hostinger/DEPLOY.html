Easy Deploy Guide for Hostinger

✅ Ready to Deploy!

This package contains everything you need. Just follow these simple steps.

1 Upload Files to Hostinger
  • Log into Hostinger hPanel
  • Go to Files → File Manager
  • Navigate to public_html/
  • Upload frontend files (all files in this folder EXCEPT backend/) directly to public_html/
  • Upload backend: create folder public_html/api/ and upload backend/ folder there
2 Create Database
  • In hPanel, go to Databases
  • Create a new MySQL database
  • Note down: Database Name, Username, Password
3 Configure Backend
  • In File Manager, go to public_html/api/
  • Find .env.example file
  • Click Edit and update these values:
DB_NAME=your_database_name
DB_USER=your_database_user
DB_PASS=your_database_password
JWT_SECRET=any-random-32-char-string
ALLOWED_ORIGINS=https://yourdomain.com
  • Save the file as .env (rename it)
⚠️ Important: Generate a strong JWT_SECRET: openssl rand -hex 32 or use any random 32-character string.
4 Import Database Schema
  • Go to hPanel → Databases → phpMyAdmin
  • Select your database
  • Click Import → Choose file
  • Upload: public_html/api/install.sql
  • Click Go
5 Set Permissions
  • In File Manager, navigate to public_html/api/
  • Find upload/ folder
  • Right-click → Change Permissions
  • Set to 755 (or check "Write" box)
6 Enable SSL (HTTPS)
  • In hPanel, go to Security → SSL
  • Enable Free SSL Certificate
  • Force HTTPS redirect
7 Test Your Site!

🔐 Security Checklist:
  • ✅ Change database password from default
  • ✅ Use strong JWT_SECRET (random 32+ chars)
  • ✅ Set correct ALLOWED_ORIGINS in .env
  • ✅ Enable SSL (HTTPS)
  • ✅ Keep .env file private (never share)
❌ Common Issues & Fixes:
IssueSolution
500 ErrorCheck .env exists and has DB credentials
CSS/JS not loadingEnsure .htaccess is in public_html/
Upload failsSet api/upload/ permissions to 755
CORS errorUpdate ALLOWED_ORIGINS with your domain
PDF parsing failsNormal on some hosts. Use simpler CVs or contact support

📞 Need Help?

  • Check error logs in hPanel → Metrics → Error Log
  • Verify all steps are completed
  • Ensure vendor/ folder exists in api/ (it should)
  • Make sure PHP version is 7.4+ (Hostinger default is fine)

🎉 You're Done!

Your CV Analyzer is now live on Hostinger!
